Registro del producto

Cada objeto del catálogo se guarda con este registro. Los campos cubren la identidad del objeto, su época, su estado y la información necesaria para la venta.

CampoTipo de datoDescripciónRol
id_productoN(6)Número único autoincremental generado por el sistema al registrar cada producto.🔑 Clave primaria
nombreAN(60)Nombre completo del objeto (ej. "Walkman Sony WM-2", "Atari 2600").—
descripcionAN(500)Historia, particularidades y estado detallado del objeto.—
categoriaAN(20)Tecnología / Audio / Gaming / Fotografía / Hogar / Otros.—
decadaN(4)Década de fabricación: 1950, 1960, 1970, 1980, 1990 o 2000.—
condicionAN(10)Estado físico: Mint (impecable) / Bueno / Regular.—
precioN(9,2)Precio de venta en pesos argentinos (ARS). Ej: 12500,00—
pais_origenAN(30)País donde fue fabricado originalmente (ej. "Japón", "EE.UU.", "Argentina").—
stockN(3)Cantidad de unidades disponibles. Generalmente 1 para piezas únicas.—
disponibleN(1)Indica si el producto puede comprarse: 1 = sí, 0 = no.—
¿Por qué id_producto es la clave?

Se eligió id_producto como clave primaria porque es un número entero autoincremental asignado automáticamente por el sistema — lo que garantiza que sea único, irrepetible y nunca vacío. Dos objetos pueden tener el mismo nombre, la misma categoría o incluso el mismo precio, pero jamás el mismo id. Esa unicidad absoluta permite identificar cada pieza sin ambigüedad, y hace que el campo funcione como referencia confiable en otros registros del sistema como órdenes de compra, historial de ventas o listas de favoritos.

Registro del proceso de trabajo con Claude para el diseño de TodoRetro.

PROMPT 01 Diseño completo de la app + estructura de datos 
Quiero hacer una tienda online de objetos vintage y tecnología retro, 
que se llame TodoRetro. La idea es que sea una app de e-commerce 
donde se puedan ver y comprar cosas como consolas viejas, cámaras 
analógicas, eso.

Necesito que me hagas la página web completa con:

* Una sección de catálogo con productos de ejemplo que se puedan 
  filtrar por categoría (tecnología, audio, gaming, fotos, etc.)

* Que cada producto tenga su nombre, categoría, década de fabricación, 
  condición (bueno/regular/etc), precio y país de origen

* Una sección que muestre la estructura de datos de los productos 
  en general, sera un solo registro, o sea que estén todos los campos 
  con su tipo de dato (texto, número, booleano, etc.) y cuál sería 
  la clave para identificar cada producto de forma única, con una 
  explicación de por qué elegiste esa clave.

* Una parte donde se vea el prompt que usé con la IA para armar 
  esto, como un registro del proceso

Para el estilo quiero algo oscuro, con onda retro/tech, que se note 
que es una tienda de objetos de otra época. Tipografía que pegue 
con eso, colores que no sean genéricos.

Que todo esté en un solo archivo HTML descargable.

// Registro definido — 10 campos:
id_producto  → N(6)      ← clave primaria (único, autoincremental)
nombre       → AN(60)
descripcion  → AN(500)
categoria    → AN(20)    (Tecnología / Audio / Gaming / Foto / Hogar)
decada       → N(4)      (1950, 1960, 1970, 1980, 1990, 2000)
condicion    → AN(10)    (Mint / Bueno / Regular)
precio       → N(9,2)
pais_origen  → AN(30)
stock        → N(3)
disponible   → N(1)      (1 = sí, 0 = no)

// Justificación de la clave:
id_producto garantiza unicidad absoluta sin importar el resto de 
los campos. Nombre, precio y categoría pueden repetirse entre 
distintos productos — el id nunca. Es generado por el sistema, 
nunca puede estar vacío ni duplicado.
PROMPT 02 Corrección de tipos de dato + cambio de nombre 
En el registro, en pseudocódigo se especifica el tipo de dato con 
otra nomenclatura:

— Para alfanumérico:  AN(n)  donde n es la cantidad máxima de caracteres
— Para numérico:      N(n)   donde n es la cantidad máxima de dígitos
                      N(n,d) si tiene decimales, d = dígitos tras la coma
                      Ejemplo: N(7,2)

Necesito que cambies los tipos de dato del registro a esa notación,
y que cambies el nombre de la app a TodoRetro.

Podés agregar este prompt al registro de interacción con la IA.
